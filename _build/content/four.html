Module 3: Introduction to the different types of databases

A database is a collection of data about entities. It can be more or less structured, depending on the type of database. It can include information about the relationships between entities. It is designed to: get and link data from multiple sources, store it efficiently so that we can analyze it easily, quickly, and efficiently and provide the results to other applications/people/systems. Most of us are used to using what’s called “flat files” with each row reflecting all the information we have on one individual over time. That structure leads to a lot of problems. Data is redundant: the same data is stored in multiple locations. Data are inconsistent because they are not all updated at the same time or in the same way. A database management system (DBMS) creates an infrastructure for storing, managing, and interacting with databases. It specifies the way the system stores data, a query language, which specifies how users interact with data, provides support for transactions and crash recovery. A good DBMS enables analysts to find where changes came from that can change the measurement of interest.
There are hundreds of different DBMS -e.g. network, hierarchical, relational, flat file model etc. The relational DBMS are the most widely used and optimal for many social science data analysis purposes. Relational databases store data records in tables, each of which has a predefined set of columns - the pieces of information captured for each record in a table. It stores data records as rows in the table, where a row has a place for a value for each column. The tables and their columns and relationships are defined in a schema.

  • 1:1 - a one to one relationship:

    for each occurrence of one entity there is exactly one occurrence of a related entity.

  • 1:m - a one to many relationship:

    for each occurrence of one entity there is one or many occurrences of a related entity. .

  • m:m - a many to many relationship:

    for each occurrence of either entity, there is one or many occurrences of the other entity

Components of a database

  • Tables: Data arranged in row-column format similar to spreadsheet.

  • Entities: Database entities can be persons, places, measurements or quantities, events, objects etc. They are fundamental to all databases and record information. Each entity has corresponding properties and are the potential tables.

  • Properties: Detailed information about the database and the potential columns.

Importance of database

Databases are the foundation of modern data management, enabling businesses, government agencies, research institutions, and various organizations to store, manage, and leverage data effectively, which, in turn, supports innovation, efficiency, and informed decision-making. We want to get data from multiple sources. Some key reasons why databases are important are as follows:

  • Data Organization: Databases provide a structured and organized way to store data. Data is categorized into tables, rows, and columns, making it easier to understand and manage.

  • Data Retrieval: Databases allow users to quickly and efficiently retrieve specific pieces of information from a vast dataset using query languages like SQL. This facilitates data analysis and reporting.

  • Concurrency Control: In multi-user environments, databases manage concurrent access to data, ensuring that multiple users can read and write data without conflicts or data corruption.

  • Data Analysis: Databases are essential for data analysis, reporting, and business intelligence. Analysts and decision-makers can use SQL queries to extract insights from data.

  • Data Sharing: Databases allow multiple users and applications to access and update data concurrently, facilitating collaboration and data sharing across different parts of an organization.

  • Data Integration: Databases can integrate data from various sources, enabling organizations to create a unified view of their data for better decision-making.

In summary, through databases support data-driven decision-making by providing accurate and up-to-date information that can inform strategic and operational choices.
